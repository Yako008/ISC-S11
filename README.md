1. Firmware: Es un software especifico que vive en el hardware, su función
principal es controlar o comunicar el circuito electrónico de un dispositivo
como el de un teclado o un microondas para que funcione sin ningún
problema.
2. SO: (Sistema Operativo) Es el software principal que administra los
recursos de hardware y permite que otros programas funcionen con
normalidad o una interfaz mas entendible. Ejemplos: Windows. Linux,
macOS, Android.
3. loT: (Internet de las Cosas) Hace referencia a la red de objetos físicos
como sensores, electrodomésticos o cámaras que tienen software y
conexión a internet para comunicarse entre sí.
4. UEFI: Es una versión más reciente de la BIOS, es la interfaz que conecta el
firmware de la computadora con el sistema operativo.
5. IaaS: (Infraestructura como servicio) Alquilan servicios básicos como
servidores virtuales, almacenamiento y redes.
6. PaaS: (Plataforma como Servicio) Te proporcionan el entorno virtual para
que solo subas tu código y funcione sin preocuparte por el servidor.
7. SaaS: (Software como Servicio) Aplicaciones listas para usar vía internet
como Google Driver o Netflix
8. APIs: Son un conjunto de reglas que permiten a aplicaciones comunicarse
entre sí.
9. DevOps: Es una cultura o metodología que busca unir el desarrollo de
software (Dev) con las operaciones de sistemas (Ops) para que el ciclo de
vida del software sea más rápido y eficiente.
10. Frontend: Es la parte virtual de una aplicación o web. Todo lo que el
usuario ve y con lo que interactúa (botones, colores, animaciones)
11. Backend: Es el procesamiento lógico oculto que se encarga de procesar
datos, manejar bases de datos y la seguridad.
12. Docker: Una herramienta que permite empaquetar una aplicación con todas
sus dependencias en un "contenedor". Esto garantiza que la app funcione
igual en cualquier computadora.
13. Cloud Engineer: El profesional encargado de diseñar, implementar y
gestionar sistemas y servicios dentro de plataformas en la nube.
14. Git: Es un sistema de control de versiones que permite a los
programadores registrar todos los cambios en el código, volver a versión
anteriores y trabajar en equipo sin sobreescribir el trabajo de otros
15. CI/CD: (Integración Continua y Despliegue Continuo) Es un practica
donde el código se prueba, se integra y se lanza a producción de forma
automática
